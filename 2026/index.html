AI創造工作坊

2026 年度課程面向台灣大專院校人文學科學生，核心不只是學會使用 AI 工具，而是成為能設計、調整、分享 AI 工作流程的 AI 創造者。課程將引導學生把閱讀、寫作、研究、翻譯、整理與創作任務，轉化為可重複、可改良、可分享的 AI-assisted workflow。

五週課程大綱：從 AI 使用者到 AI 工作流程創造者

本課程以人文學科的學習、研究與創作需求為出發點，帶領學生理解 AI agent、tools、AGENTS.md、skills、SKILL.md、MCP、Git/GitHub、GitHub Pages、Obsidian 與 Markdown 如何共同構成可操作的個人 AI 工作流程。許多人在使用現成 AI 工具時，會覺得輸出不夠貼近自己的需求；問題未必在於 AI 不夠強，而是缺少能引導它理解任務、脈絡與步驟的技能設計。完成課程後，學生將能把一個日常學習或創作任務，整理為具備專案脈絡、操作規則、技能規格與公開展示頁的 personal AI-assisted workflow，讓 AI 逐步配合自己的閱讀、寫作與研究方法，而不是被既有軟體功能牽著走。課程的學習目標是對相關技術達到「夠用的掌握」，而不需要深度精通；這呼應 Anthropic的使用者專業能力的研究報告 所指出的趨勢：有效使用 AI 工具時，重點往往在於能理解脈絡、判斷方向並組織工作流程。

  • 第 1 週：認識 AI Agents：把 agent 想成一位會讀脈絡、能照著任務清單行動的研究助理，不只是回答問題的聊天對象。
  • 第 2 週：認識 tools、CLI、GUI：理解工具就像研究者的筆、索引卡與資料庫；CLI 像直接下指令的工作台，GUI 則像有按鈕與選單的書桌介面。
  • 第 3 週：認識 AGENTS.md 與 SKILL.md：前者像寫給助理的研究室規範，後者像一張可重複使用的工作方法卡。
  • 第 4 週：認識 MCP：把 MCP 想成 AI agent 與外部工具、資料來源之間的翻譯與接線員，讓不同系統能彼此溝通。
  • 第 5 週：認識 GitHub Pages：把它想成自己的線上展覽牆，讓學習歷程、研究流程與創作成果可以被看見與分享。

2026 課程地圖

W1

認識 AI Agents

把 AI agent 想成一位研究助理：它不只陪你聊天，也能讀懂任務、參考脈絡，並依照步驟協助整理材料。

W2

認識 tools、CLI、GUI

Tools 像研究工具箱；CLI 像直接寫給工具的指令紙條，GUI 則像有抽屜、按鈕與標籤的書桌。

W3

認識 AGENTS.md、SKILL.md

AGENTS.md 像研究室守則，SKILL.md 像方法筆記，幫 AI 記住該用什麼口吻、規則與流程工作。

W4

認識 MCP

MCP 像圖書館的流通櫃台與翻譯員，讓 AI agent 能連接不同資料庫、工具與服務。

W5

認識 GitHub Pages

GitHub Pages 像線上展覽空間，讓你的學習歷程、研究流程與數位作品有一個可以分享的公開入口。
